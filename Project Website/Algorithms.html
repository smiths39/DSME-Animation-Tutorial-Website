Algorithms

What is an Algorithm?

An algorithm is a set of instructions that a computer follows in order to accomplish a specific task required. These tasks can include sorting a set of numbers or finding the greater common denominator of two numbers.

Problem Areas

In our experience, and from research carried out by us, we find that Quicksort is regarded as the most essential algorithm to understand as it is often implemented upon a majority of applications.

We have put a major focus on Quicksort, and other problem areas, to ensure that understanding its functionality is as painless as possible.

You can read our full tutorial on Quicksort here. It includes diagrams, pseudocode and interactive animations which will give you an advance interpretation on how it performs. Also, be sure to watch a screencast on Quicksort, which includes how you would approach implementing it in a Java program.

We believe it is important that you don't dive into the deep end first, make sure you're comfortable with the most common algorithms before expanding your research on more complex algorithms. See the Approach section below to find out the best way to tackle learning them.

Approach

As previously stated, it's important not to dive into the deep end first. Take your time mastering each algorithm before you move on.

Firstly, start off with Bubble Sort and understand its use and implementation.

It is best to start off with this algorithm as its implementation is relatively easy.

After you're comfortable with how Bubble Sort works and if you can personally implement it, then progress to other common algorithms, such as Insertion Sort, Selection Sort, Merge Sort and Heap Sort.

Lastly, move on to Quicksort. As it is considered to be the most common algorithm to implement, due its optimal complexity rate. It is important that you have a good understanding of these algorithms as they frequently arise as the best candidates to implement within certain applications.
