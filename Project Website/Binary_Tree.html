Binary Search Tree

Definition

A Binary Search Tree (BST) is essentially a collection of nodes. Each node contains a value and a link to the next node in the tree (either to the left or right). There are no duplicate nodes in a BST.

In the example below there is one central node, called the root node. The value of all nodes to the left of the root are less than its value. E.g. 11 is less than 15. The value of all nodes to the right of the root are greater than its value. E.g. 21 is greater than 15.

When a child node has its own children, the same rules apply. Any node to the left will be less than the parent node. Any node to the right will be greater. As you can see in the example below, 5 is less than 11, but 13 is greater than 11.

Advantages

When the tree is balanced, inserting and removing elements is very fast.

If you're at any particular node, finding the next one is easy as all child nodes are connected to their parent.

Disadvantages

The tree can lose its shape depending on the order in which the elements are added.
