Queue

Definition

Like a stack, a queue is a non-linear data structure for storing data. Unlike the stack, a queue operates on a First-In-First-Out basis.

Imagine a queue of people at a bank. The person at the front of the queue will be seen by the bank teller first. Therefore, they will be the first person to leave the bank. So when using a queue in a program, when you remove an element, the element at the front of the list will be removed first.

Again, similar to a stack, the queue comes in two forms - Bounded and Unbounded.

A Bounded queue is one which has a maximum size limit, i.e. there is only a limited number of elements which can be added to it.

An Unbounded queue is one which doesn't have a maximum size limit, i.e. the queue will grow arbitrarily in size when it needs to cater for more elements.

Example

  • In the example above, the number 1 is added or "enqueued" onto the queue.
  • We then enqueue two more elements, the numbers 2 and 3 respectively.
  • As displayed above, the first element, number 1, is always at the front of the queue.
  • When we want to remove an element from the queue, we call "dequeue".

Because a queue operates on a First-In-First-Out basis, the element to be removed will be the first element that was added in.

Complexity

The only considerations to be made in relation to the complexity of a queue are the operations it performs. The operations of enqueue and dequeue possess a time complexity of O(1) on average.

Advantages

A queue is relatively easy to implement as it only requires a limited number of operations - primarily, add and remove.
As queue operates on a First-In-First-Out basis, it is advantageous for prioritising selected data.

Disadvantages

A queue is considered to be of minimal use when compared to other optimal data structures. Based on the structure of queues, they suffer from slow access to other items, which can increase the computational time of the application.

Implementation

As we mention above, implementing a Queue is relatively easy as there are a limited amount of operations.
