Stack

Definiton

A stack is a non-linear data structure which facilitates the addition and removal of data. It operates on a Last in, First out basis.

For example, if you have the the following sequence of integers - 1,2,3,4,5 - and you add them to a stack in that precise order, upon the decision to remove an element - the number 5 (last element inserted), will be the first element removed.

Similiar to Queues, a Stack come in two forms - bounded and unbounded.

A bounded stack is one which has a maximum size limit. It contains a sequence of elements which increment or decrement at one end of the stack only.

An unbounded stack is the same as above, except it has no maximum size. It can grow to cater for as many inputs as possible.

Complexity

The only considerations to be made in relation to the complexity of a stack are the operations it performs. The operations of pushing and popping possess a time complexity of O(1).

Advantages

Compared to other data structures, it is easily implemented because there are limited operations which can be invoked. A stack has a low requirement for computer hardware, which serves as a strong advantage whilst taking into consideration. The accessibility range of stack is another consideration, as anyone who has access can edit the program.

Disadvantages

A stack is considered to be of minimal use when compared to other optimal data structures. The inflexibility of a stack and the lack of scalability have deteriorated its use in various applications.

Implementation
